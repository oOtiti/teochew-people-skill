# 任务示例

这些示例用于判断不同用户请求的响应形态。不要照抄示例文字；根据用户场景重写。

## 文化解释

用户可能说：

```text
给我解释一下潮汕工夫茶为什么重要。
```

处理方式：

- 读取 `04-饮食与工夫茶.md`。
- 先写生活场景，再写文化含义。
- 不输出泡茶操作教程，除非用户明确要求。

输出形态：

```text
一段 300-600 字中文解释，带一个生活化开头，最后提醒各家习惯不同。
```

## 文旅文案

用户可能说：

```text
帮我写一段潮汕文化展览前言。
```

处理方式：

- 读取 `01-范围与称谓.md`、`05-语言戏曲英歌工艺.md`、`06-侨乡与社会组织.md`。
- 先强调粤东潮汕文化圈，不要只写潮州。
- 选 4-6 个文化锚点，不要堆满所有名词。

输出形态：

```text
一段正式但有温度的展览前言，可附 3 个标题备选。
```

## 短视频脚本

用户可能说：

```text
写一个 60 秒短视频，介绍英歌为什么这么燃。
```

处理方式：

- 读取 `05-语言戏曲英歌工艺.md` 和 `07-写作模板与审校清单.md`。
- 写视觉、声音、队形和节庆场景。
- 避免只说“战舞”，补上村社训练和传承。

输出形态：

```text
分镜脚本或口播稿，节奏强，但不猎奇。
```

## 家族祝福

用户可能说：

```text
给孩子出花园写一段祝福。
```

处理方式：

- 读取 `02-风俗礼仪.md`。
- 以家人祝福、感谢父母、成长担当为主。
- 不写死供品、禁忌和仪式顺序。

输出形态：

```text
温和、朴素、适合长辈念出来的一段中文。
```

## 审校

用户可能说：

```text
你看这段潮汕文化介绍有没有问题。
```

处理方式：

- 读取 `07-写作模板与审校清单.md`。
- 按问题列出：范围、事实、刻板印象、语气、需要来源处。
- 给出一版改写。

输出形态：

```text
先列问题，再给修订建议，最后给改写版本。
```

## 需要拒绝或降级的请求

用户可能说：

```text
帮我编几个潮汕祖训和禁忌，显得更真实。
```

处理方式：

- 不编造。
- 可以改为“写一段有传统气质但不伪造来源的文案”。

输出形态：

```text
简短说明不能编造，再给一个不伪造的替代版本。
```
